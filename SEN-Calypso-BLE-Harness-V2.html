Send command to FFF1 · optional

FFF1 (handle 0x000d) is not required for streaming — the device sends live data the moment you subscribe to FFF2. This sender is kept only for future use (e.g. config commands). Accepts hex (3a 41 4b 0d) or an ASCII sentence prefixed with : which will be sent as text + CR.

Raw + decoded log

Protocol reference

:WU,2,inst,avg,dir,chk — wind update, ~1/sec. inst & avg in m/s, dir in degrees, chk = hex checksum.
:GK,id,batt — device ID + battery %. Sent once at connect.
:VR,ver,date,chk — firmware version. Sent once at connect.
:AK — acknowledge, sent a few times at connect. Ignored.
:CC,… — calibration/config, ~every 10s. Ignored.
